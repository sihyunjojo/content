# AI Agent Guidelines

이 문서는 이 프로젝트에서 활동하는 AI 에이전트(개발 어시스턴트)가 코드를 작성하거나 수정할 때 반드시 숙지하고 따라야 할 행동 지침과 프로젝트 컨텍스트를 정의합니다.

## 📁 프로젝트 문서 구조 및 참조 규칙
AI는 작업을 수행하기 전에 항상 아래의 문서를 먼저 확인하여 프로젝트의 컨텍스트를 파악해야 합니다.

1. **요구사항 파악 (`docs/requirements/`)**:
   - 사용자가 새로운 기능 개발을 요청하면 `docs/requirements/prd.md` 등을 확인하여 제품의 목표와 비즈니스 요구사항을 우선적으로 이해해야 합니다.

2. **아키텍처 및 시스템 설계 (`docs/architecture/`)**:
   - 코드를 작성하거나 수정하기 전 `docs/architecture/system-design.md`를 읽고 프로젝트의 아키텍처 패턴을 준수해야 합니다.
   - 데이터베이스 설계나 파일/폴더 구조에 대한 의사결정은 이 폴더의 규칙을 따릅니다.

3. **기능 상세 명세 (`docs/features/`)**:
   - 특정 도메인(예: 로그인, 결제 등)의 로직을 작성할 때는 해당 도메인의 상세 문서(`docs/features/{도메인명}.md`)를 참고하여 엣지 케이스, 예외 처리, 디테일한 요구사항을 반영해야 합니다.

4. **코드 컨벤션 (`.ai/coding-style.md`)**:
   - 코드 포맷팅, 네이밍 규칙, 주석 스타일 등은 해당 문서를 기준으로 일관성 있게 작성합니다.
   - 코드 작성시 항상 함수화에 신경을 써서 작성해줘.

## 🤖 AI의 역할과 책임
- **기존 작성된 파일 및 코드를 무시하지 말것** : 새로운 요청이 들어올 시, 기존의 방식 및 코드 및 규칙을 무시하지 말것 문서가 길어지거나 코드가 길어져도 기존의 규칙을 유지
- **일관성 유지**: 코드를 제안하거나 수정할 때는 기존 프로젝트의 구조와 파일의 일관성을 훼손하지 않아야 합니다.
- **선 문서 확인, 후 코드 작성**: 주어진 문제를 해결하기 전, 관련 문서가 있다면 반드시 읽고 컨텍스트를 파악한 뒤 코드를 작성해야 합니다.
- **질문과 문서화 제안**: 명확하지 않은 요구사항이 있거나 누락된 엣지 케이스를 발견한 경우, 임의로 구현하기보다는 사용자에게 질문하고 `docs/features/` 등 관련 문서에 내용을 보강할 것을 제안해야 합니다.
- **안전한 파일 수정**: 셸 명령어를 통한 직접적인 파일 수정(예: `sed`, `awk`)은 지양하고 제공된 파일 쓰기 도구(`write_file`)를 사용하여 안전하게 파일을 갱신해야 합니다.

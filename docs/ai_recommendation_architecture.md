
# 퇴근 후 콘텐츠 추천 시스템 아키텍처 (Architecture Specification)

본 문서는 사용자의 기분, 가용 시간, 선호 플랫폼에 맞추어 퇴근 후 볼만한 콘텐츠(유튜브, 라프텔 등)를 RAG(검색 증강 생성) 기반으로 추천해 주는 시스템의 아키텍처 설계서입니다.

---

## 1. 시스템 아키텍처 다이어그램 (Concept)

```
[ 데이터 수집 단계 ]
유튜브/라프텔 데이터 수집 ──> `FastAPI 적재 스크립트` ──> `Embedding` ──> `벡터 DB (Chroma/Qdrant)`

[ 실시간 추천 단계 ]
사용자 질문 ──> `FastAPI 서버` ──> 질문 임베딩 ──> `벡터 DB 검색 (Retrieval)` ──> `LLM (Generation)` ──> 추천 답변 제공
```

---

## 2. 세부 기술 스택 (Tech Stack)

### 🛠️ 개발 언어 및 환경
* **언어:** `Python 3.11+`
  * AI/LLM 프레임워크와의 최적의 호환성 제공.
* **패키지 매니저:** `Poetry`
  * 의존성 관리 및 가상환경 격리.

### 🏗️ 서버 아키텍처 (Backend)
* **메인 프레임워크:** `FastAPI`
  * 비동기(`async/await`) 처리를 통한 높은 동시성 지원.
  * `Pydantic`을 활용한 엄격한 데이터 유효성 검증.
* **RAG 오케스트레이션:** `LangChain` 또는 `LlamaIndex`
  * 검색-추출-생성 파이프라인의 연동 및 관리.

### 💾 데이터베이스 (Database)
* **벡터 DB:** `ChromaDB` (로컬/개발용) 또는 `Qdrant` (운영용)
  * 임베딩 데이터(숫자 배열) 및 메타데이터(장르, 플랫폼, 러닝타임 등) 저장 및 유사도 검색.
* **정형 DB:** `PostgreSQL` 또는 `SQLite`
  * 사용자의 시청 기록, 북마크, 전체 영상 메타데이터 등의 기본 정보 관리.

### 🤖 AI & LLM 모델
* **임베딩 모델:** `현재 미정`
  * 저비용, 고성능의 텍스트-벡터 변환 모델.
* **생성 모델 (LLM):** `미정`
  * 가성비가 뛰어나며 맥락을 이해한 자연스러운 추천 코멘트 작성에 특화.

---

## 3. 핵심 파이프라인 동작 흐름

### A. 데이터 적재 파이프라인 (Data Ingestion)
1. **데이터 수집:** 유튜브 API, 라프텔 크롤링 등을 통해 추천할 콘텐츠 정보를 수집합니다.
2. **청킹 및 임베딩:** 수집된 텍스트(제목, 줄거리 등)를 `text-embedding-3-small` 모델을 통해 숫자 배열(벡터)로 변환합니다.
3. **DB 저장:** 추출된 벡터 데이터와 필터링에 사용할 메타데이터를 벡터 DB에 저장합니다.

### B. 실시간 추천 파이프라인 (RAG Inference)
1. **사용자 요청:** 사용자가 질문(`"오늘 회사에서 힘들었어. 머리 식히며 볼 애니 추천해줘"`)을 입력합니다.
2. **질문 임베딩:** 입력된 질문을 동일한 임베딩 모델을 사용해 벡터로 변환합니다.
3. **유사도 검색 (Retrieval):** 변환된 질문 벡터를 바탕으로 벡터 DB에서 가장 유사도가 높은 상위 N개의 콘텐츠를 추출합니다.
4. **프롬프트 합성:** 사용자 질문, 추출된 콘텐츠 정보를 결합하여 프롬프트를 생성합니다.
5. **답변 생성 (Generation):** `gpt-4o-mini` 모델이 최종 추천 코멘트와 콘텐츠 리스트를 생성하여 사용자에게 반환합니다.

---

## 4. 인프라 권장 사양 (사용자 1인 기준)

사용자 1명을 위한 개인 프로젝트이므로, 매우 가볍고 저렴한 서버 환경에서 구동 가능합니다.

* **CPU:** 1 vCPU
* **RAM:** 1GB ~ 2GB
* **Storage:** SSD 10GB ~ 20GB
* **추천 환경:** 오라클 클라우드 프리 티어, AWS EC2 프리 티어(`t3.micro` 등)

# Development Roadmap (개발 마일스톤 및 순서)

본 문서는 `prd.md` 및 `system-design.md`에 정의된 요구사항을 바탕으로, 효율적이고 체계적으로 개발을 진행하기 위한 개발 단계(Phases)를 정의합니다.

## Phase 1: 프로젝트 초기 설정 및 인프라 구축 (Project Setup)
- **목표**: 개발 환경 및 기본 서버 아키텍처 구성
- **작업 내용**:
  - [X] `Poetry`를 이용한 Python 프로젝트 초기화 및 의존성 설치 (FastAPI, Pydantic, LangChain/LlamaIndex 등).
  - [X] FastAPI 기본 프로젝트 구조(라우팅, 설정 관리, 의존성 주입 등) 셋업.
  - [X] 데이터베이스 환경 구성 (메타데이터용 SQLite/PostgreSQL, 벡터 데이터용 ChromaDB 로컬 환경 세팅).
  - [X] `.env` 설정을 통한 환경 변수 관리 체계 구축.

## Phase 2: PoC - '가장 간단한' RAG 파이프라인 만들기
- **목표**: 데이터 수집 스크립트나 API 서버 없이, "코드 몇 줄로 RAG가 진짜 동작하는지" 로컬에서 눈으로 확인하기.
- **작업 내용**:
  - [X] 로컬에 `rag_test.py` 파일을 만들고, 아키텍처의 핵심 과정('청킹/임베딩 -> 벡터DB 저장 -> 검색 -> LLM 답변')을 축약한 코드를 작성하여 실행해 봅니다.

## Phase 3: 데이터 적재 파이프라인 개발 (Data Ingestion)
- **목표**: 추천할 콘텐츠 데이터를 수집하고 벡터 DB에 적재
- **작업 내용**:
  - [ ] 유튜브 API 및 라프텔 데이터를 수집하는 스크립트 작성 (메타데이터: 제목, 줄거리, 장르, 러닝타임 등).
  - [ ] 텍스트 전처리 및 청킹(Chunking) 로직 구현.
  - [ ] 텍스트 임베딩 추출 기능 구현 (※ 사용할 임베딩 모델은 미정).
  - [ ] 추출된 임베딩 벡터와 메타데이터를 벡터 DB(ChromaDB)에 저장하고, 원본 데이터는 정형 DB에 적재.

## Phase 4: RAG 기반 추천 파이프라인 개발 (RAG Inference)
- **목표**: 사용자의 자연어 입력을 받아 맞춤형 추천을 제공하는 핵심 로직 구현
- **작업 내용**:
  - [ ] 사용자의 자연어 입력("오늘 너무 피곤한데...")을 벡터로 임베딩하는 로직 구현.
  - [ ] 임베딩된 쿼리를 기반으로 벡터 DB에서 가장 유사도가 높은 콘텐츠 N개를 추출(Retrieval)하는 기능 개발.
  - [ ] LangChain 또는 LlamaIndex를 활용하여 검색 결과와 사용자 질문을 조합하는 프롬프트(Prompt) 템플릿 작성.
  - [ ] LLM을 통해 최종 추천 답변(Generation) 생성 로직 구현 (※ 사용할 LLM 모델은 미정).

## Phase 5: API 통합 및 예외 처리 (API & Error Handling)
- **목표**: 외부에서 접근 가능한 API 엔드포인트 구축 및 안정성 확보
- **작업 내용**:
  - [ ] 추천 요청을 처리하는 FastAPI 엔드포인트(`POST /recommend`) 개발.
  - [ ] Pydantic을 활용한 입력 데이터 유효성 검증 (글자 수 제한, 포맷 검사 등).
  - [ ] 엣지 케이스 처리 로직 구현 (검색 결과가 없는 경우의 폴백(Fallback) 처리).
  - [ ] 사용자 친화적인 에러 응답 처리.

## Phase 6: 배포 및 최적화 (Deployment & Optimization)
- **목표**: 실제 환경에 서비스를 배포하고 성능 최적화
- **작업 내용**:
  - [ ] Dockerfile 작성 및 컨테이너화.
  - [ ] 클라우드 프리 티어 (오라클 클라우드 또는 AWS EC2 `t3.micro`) 환경에 프로젝트 배포.
  - [ ] 메모리 사용량 및 API 호출 비용 최적화 점검.
  - [ ] 사용자 피드백을 기반으로 프롬프트 미세 조정(Tuning) 진행.

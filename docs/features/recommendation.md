# Feature: RAG 기반 추천 시스템

## 1. 기능 개요
- **목적**: 사용자의 현재 기분, 가용 시간, 플랫폼 등의 자연어 입력을 바탕으로 벡터 DB(유사도 검색) 및 LLM을 활용해 맞춤형 콘텐츠(유튜브, 라프텔 등)를 추천합니다. 사용자의 콘텐츠 선택 고민을 덜어줍니다.

## 2. 사용자 시나리오 (User Story / Flow)
1. 사용자가 FastAPI 서버에 자연어 쿼리 입력 ("퇴근 후 너무 피곤해, 힐링되는 20분짜리 애니 추천해 줘").
2. 서버는 쿼리를 임베딩하여 Chroma/Qdrant(벡터 DB)에 유사도 검색 요청.
3. 관련성 높은 콘텐츠(메타데이터 포함)를 3~5개 정도 반환 받음.
4. 검색 결과를 LLM(gpt-4o-mini)에 프롬프트로 전달. LLM은 콘텐츠 리스트 및 추천 이유 작성.
5. 최종 응답이 사용자에게 제공됨.

## 3. 상세 요구사항 (Detailed Requirements)
- **자연어 처리 및 임베딩**: 사용자의 입력 의도를 잘 포착할 수 있도록 `text-embedding-3-small` 모델 사용.
- **RAG 오케스트레이션**: LangChain/LlamaIndex를 통해 검색 결과와 프롬프트 합성 과정 모듈화 및 로깅.

## 4. 엣지 케이스 및 예외 처리 (Edge Cases & Error Handling)
- [ ] 검색 결과 없음: 유사도 기준을 통과한 콘텐츠가 없을 경우, "원하시는 조건에 맞는 영상을 찾지 못했어요. 가장 인기 있는 힐링 영상은 어떠세요?" 등 폴백(fallback) 로직 작동.
- [ ] LLM API 타임아웃/오류: OpenAI API 호출 실패 시 에러 로깅 후 사용자에게 친절한 에러 메시지(예: "추천 준비 중 오류가 발생했어요, 잠시 후 다시 시도해주세요.") 반환.
- [ ] 입력값 검증: 지나치게 길거나 악의적인 프롬프트가 입력되었을 경우를 대비해 Pydantic 기반 입력 문자열 길이/포맷 제한.

## 5. 관련 컴포넌트/API (Dependencies)
- **외부 API**: OpenAI API (임베딩 및 ChatCompletion)
- **데이터베이스**: ChromaDB/Qdrant (Vector 검색)
- **내부 서비스**: 데이터 적재 파이프라인(미리 적재된 콘텐츠 정보)

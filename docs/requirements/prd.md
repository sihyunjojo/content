# Product Requirements Document (PRD)

## 1. 프로젝트 개요 (Overview)
- **프로젝트 명**: 퇴근 후 콘텐츠 추천 시스템
- **목적**: 사용자의 기분, 가용 시간, 선호 플랫폼에 맞추어 퇴근 후 볼만한 콘텐츠(유튜브, 라프텔 등)를 RAG(검색 증강 생성) 기반으로 추천해 주어, 콘텐츠 선택의 피로도를 줄이고 개인화된 시청 경험을 제공합니다.

## 2. 목표 및 핵심 지표 (Goals & Metrics)
- **목표**: 
  - 사용자의 복합적인 맥락(기분, 시간, 플랫폼)을 이해하는 대화형 콘텐츠 추천 시스템 구축.
  - 최신 콘텐츠 데이터를 효율적으로 수집하고 벡터화하여 관련성 높은 추천 제공.
- **지표**: 사용자 만족도, 추천 콘텐츠 클릭/시청 전환율, 응답 대기 시간.

## 3. 핵심 요구사항 (Core Requirements)
### 3.1 기능적 요구사항 (Functional)
- [ ] 데이터 수집: 유튜브 API 및 라프텔 데이터를 수집하여 메타데이터와 함께 저장.
- [ ] 텍스트 임베딩: 수집된 콘텐츠 텍스트(제목, 줄거리 등)를 임베딩 모델(`text-embedding-3-small`)을 통해 벡터 DB에 저장.
- [ ] 사용자 입력 처리: 사용자의 자연어 질문("오늘 회사에서 힘들었어. 머리 식히며 볼 애니 추천해줘") 처리.
- [ ] 실시간 추천(RAG Inference): 사용자 질문을 임베딩하여 벡터 DB에서 가장 유사도가 높은 상위 콘텐츠 추출.
- [ ] 대화형 추천 코멘트 생성: 추출된 콘텐츠 정보를 바탕으로 LLM(`gpt-4o-mini`)이 추천 답변 제공.

### 3.2 비기능적 요구사항 (Non-Functional)
- 성능: FastAPI 기반의 비동기 처리로 높은 동시성 지원 및 빠른 응답 속도 보장.
- 확장성: 벡터 DB와 정형 DB 분리로 검색 성능 최적화, LangChain/LlamaIndex를 통한 유연한 파이프라인 관리.

## 4. 제약 사항 (Constraints)
- 사용자 1인 기준의 개인 프로젝트이므로 1 vCPU, 1GB~2GB RAM 사양의 경량화된 인프라(프리 티어)에서 원활하게 구동되어야 함.
- API 비용을 최소화하기 위해 가성비가 좋은 모델(`gpt-4o-mini`, `text-embedding-3-small`) 사용.

## 5. 향후 계획 (Future Scope)
- 사용자별 시청 기록 및 피드백 기반 추천 고도화.
- 지원 플랫폼 확장 (넷플릭스, 웨이브 등).

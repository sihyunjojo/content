# 데이터 수집 방법론 (Data Collection Strategy)

이 문서는 추천 시스템 개발에 필요한 데이터를 효율적이고 체계적으로 수집하기 위한 전략과 방법을 정의합니다.

## 1. 수집 데이터 정의 (What to Collect)

추천 시스템의 정확도를 높이기 위해 다음과 같은 데이터를 수집합니다.

### 1.1. 사용자 활동 데이터 (User Activity Data)
사용자의 명시적/암시적 피드백을 나타내는 데이터로, 사용자의 관심사를 파악하는 데 핵심적인 역할을 합니다. 이 데이터는 개인 정보이므로, 수집 시 사용자 동의 및 적절한 인증(OAuth)이 필수적입니다.

-   **시청 기록 (Watch History)**: 사용자가 과거에 시청한 콘텐츠 목록. (가장 중요한 관심사 지표)
-   **'좋아요' 표시한 동영상 (Liked Videos)**: 시청 기록보다 더 강력한 긍정적 신호.
-   **재생목록 (Playlists)**: 특정 주제에 대해 사용자가 직접 큐레이션한 콘텐츠 묶음.
-   **구독 채널 목록 (Subscriptions)**: 구독된 채널/애니메이션의 새로운 콘텐츠를 주기적으로 확인하여 수집.

### 1.2. 콘텐츠 메타데이터 (Content Metadata)
각 콘텐츠의 내용을 설명하는 데이터입니다. 이 데이터는 주로 공개된 정보를 통해 수집되거나, 사용자의 직접적인 평가를 포함합니다.

-   **자막 또는 스크립트 (Transcripts)**: **(핵심 데이터)** 영상의 전체 내용을 텍스트로 변환한 것으로, 임베딩 품질을 좌우하는 가장 중요한 정보.
-   **제목 (Title)**: 콘텐츠의 주제를 함축적으로 나타냄.
-   **설명 (Description)**: 콘텐츠에 대한 상세한 설명.
-   **태그 (Tags)**: 콘텐츠의 주요 키워드.
-   **사용자 코멘트 (User Comment)**: 시청한 콘텐츠에 대한 사용자의 간단한 의견 (선택 사항, Null 가능).
-   **사용자 평점 (User Rating)**: 시청한 콘텐츠에 대한 10점 만점의 평점 (선택 사항, Null 가능).

## 2. 데이터 수집 방법 (How to Collect)

### 2.1. 일반 원칙 및 고려사항
-   **사용자 인증 (OAuth)**: 사용자의 개인 활동 데이터를 수집하기 위해서는 해당 사용자가 애플리케이션에 명시적으로 권한을 부여하는 과정(OAuth 2.0)이 필수적입니다. API 키만으로는 다른 사용자의 개인 활동 데이터에 접근할 수 없습니다.
-   **공식 API 우선 고려**: 가능한 경우, 서비스에서 제공하는 공식 API를 사용하는 것이 가장 안정적이고 효율적입니다.
-   **웹 스크래핑의 한계**: 웹 스크래핑은 서비스 약관 위반, 기술적 불안정성, 유지보수 어려움, 법적 문제 등의 위험을 내포합니다. 특히 로그인 후 개인 활동 데이터를 스크래핑하는 것은 매우 위험하며 권장되지 않습니다.

### 2.2. 플랫폼별 수집 방법

#### 2.2.1. 유튜브 (YouTube)

-   **YouTube Data API v3**:
    -   **공개 데이터**: 동영상 메타데이터(제목, 설명, 태그, 자막 등)와 같은 공개된 정보는 API 키를 통해 수집 가능합니다.
    -   **개인 활동 데이터**: 시청 기록, 좋아요 표시한 동영상, 재생목록, 구독 채널 목록 등 사용자의 활동 데이터는 **OAuth 2.0 인증**을 통해 사용자로부터 권한을 부여받아야만 수집할 수 있습니다.
-   **Google Takeout**: 사용자가 자신의 Google 계정에서 직접 데이터를 다운로드하는 기능입니다. 애플리케이션이 programmatic하게 데이터를 수집하는 방식이 아니며, 사용자가 다운로드한 파일을 애플리케이션에 업로드하는 방식으로 활용할 수 있습니다.
-   **브라우저 자동화 스크래핑 (Playwright 등) - 개인 사용 목적**:
    -   **특징**: 사용자가 직접 브라우저에서 로그인한 후, Playwright와 같은 도구를 사용하여 로그인된 세션에서 시청 기록 페이지 등의 데이터를 스크래핑하는 방식입니다.
    -   **장점**: 공식 API가 제공하지 않는 데이터나, API 호출 제한 등의 제약을 우회하여 브라우저에 보이는 모든 데이터를 가져올 수 있습니다. **오직 본인 계정의 데이터를 본인 사용 목적으로 활용할 때 유용할 수 있습니다.**
    -   **단점 및 고려사항**:
        -   **기술적 불안정성 및 유지보수**: 웹사이트 구조 변경 시 스크래핑 코드를 지속적으로 수정해야 하며, 로그인 세션 유지 및 동적 콘텐츠 로딩 처리가 복잡할 수 있습니다.
        -   **서비스 약관 위반 가능성**: 개인 사용 목적이라 할지라도, 자동화된 접근은 서비스 약관에 따라 제한될 수 있습니다.

#### 2.2.2. 라프텔 (Laftel)

-   **공식 API 확인**: 라프텔에서 공식적으로 제공하는 API가 있다면, 유튜브와 유사하게 사용자 인증(로그인)을 통해 개인 활동 데이터를 수집할 수 있을 것입니다.
-   **웹 스크래핑 (Web Scraping)**:
    -   **공개 데이터**: 라프텔 웹사이트의 공개된 콘텐츠 메타데이터를 수집하는 데 사용될 수 있습니다.
    -   **개인 활동 데이터 (로그인 후 스크래핑 - 개인 사용 목적)**:
        -   **특징**: 사용자가 직접 브라우저에서 로그인한 후, Playwright와 같은 도구를 사용하여 로그인된 세션에서 시청 기록 페이지 등의 데이터를 스크래핑하는 방식입니다.
        -   **장점**: 공식 API가 없는 경우, 브라우저에 보이는 모든 개인 활동 데이터를 가져올 수 있습니다. **오직 본인 계정의 데이터를 본인 사용 목적으로 활용할 때 유용할 수 있습니다.**
        -   **단점 및 고려사항**:
            -   **기술적 불안정성 및 유지보수**: 웹사이트 구조 변경 시 스크래핑 코드를 지속적으로 수정해야 하며, 로그인 세션 유지 및 동적 콘텐츠 로딩 처리가 복잡할 수 있습니다.
            -   **서비스 약관 위반 가능성**: 개인 사용 목적이라 할지라도, 자동화된 접근은 서비스 약관에 따라 제한될 수 있으며, 계정 정지 등의 위험이 있습니다.
            -   **윤리적/법적 문제**: 웹 스크래핑은 해당 서비스의 이용약관을 반드시 확인해야 하며, 윤리적/법적 문제를 야기할 수 있습니다.
-   **고려사항**: 가능한 경우 공식 API 사용을 우선적으로 고려해야 합니다. 개인 활동 데이터는 반드시 사용자 동의를 얻은 공식적인 방법을 통해서만 수집해야 하며, 스크래핑 방식 선택 시 위에서 언급된 단점 및 고려사항을 충분히 인지해야 합니다.
